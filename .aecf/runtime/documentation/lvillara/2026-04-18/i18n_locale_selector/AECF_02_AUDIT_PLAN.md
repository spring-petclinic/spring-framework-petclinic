### D4 — Warnings

#### ⚠️ WARN-1: `messages_en.properties` intentionally empty

The plan correctly leaves `messages_en.properties` empty (per its existing comment: "falls back to default"). The 4 new keys in `messages.properties` serve as the English texts. This is correct behavior.

#### ⚠️ WARN-2: `pageContext.request.requestURI` strips existing query parameters

If a page already has a query parameter (e.g., `/owners?lastName=Smith`), the lang switch link `${pageContext.request.requestURI}?lang=es` will produce `/owners?lang=es`, dropping `?lastName=Smith`. This is acceptable for a demo application. The query string loss is a known limitation of the simple URI approach.

#### ⚠️ WARN-3: i18n coverage is minimal

Only 8 validation/error keys are translated. The UI labels (button text, form labels, navigation labels) remain hardcoded in JSP. This is pre-existing technical debt, out of scope for this feature.
