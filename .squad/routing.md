# Routing Rules

## Domain Routing

| Signal | Route To |
|--------|----------|
| Lab structure, scope, review requests | Bartlet |
| Step-by-step content, documentation, lab writing | CJ |
| Teaching moments, learning arcs, pedagogy | Toby |
| AI patterns, app modernization, technical depth | Josh |
| Multi-domain lab work | CJ + Toby (parallel), Josh for technical review |

## Review Gates

| Artifact | Reviewer | Gate |
|----------|----------|------|
| Lab markdown files | Bartlet | Must approve before final |
| Teaching moment sections | Toby | Advisory review |
| Technical accuracy | Josh | Must verify AI/modernization claims |

## Escalation

- Ambiguous routing → Bartlet decides
- Scope questions → Bartlet + user
- Technical accuracy disputes → Josh has final say on domain content
