# AECF — TEMPLATE HEADERS

> **SINGLE SOURCE OF TRUTH** for the mandatory metadata block that ALL AECF-generated documents must include.
>
> All templates reference this file. If the metadata standard changes, update ONLY this file.

------------------------------------------------------------

## PURPOSE

Every document generated by AECF MUST include a `## METADATA` section as its **LAST section** (at the end of the document) in compact format. This ensures:

- **Traceability**: Every document is linked to its origin prompt/skill
- **Accountability**: Execution identity fields identify who triggered the generation and from where
- **Auditability**: Sequential numbering and cross-references enable reconstruction
- **Executive visibility**: Any document can be contextualized at a glance

------------------------------------------------------------

## STANDARD METADATA BLOCK (MANDATORY FOR ALL DOCUMENTS)

When generating ANY AECF document, the AI MUST insert this compact block as the **last section** of the document, filling in all values:

```markdown
## METADATA

Document Type={{DOCUMENT_TYPE}}; AECF Number=AECF_[NN]; TOPIC={{TOPIC}}; Scope={{SCOPE}}; Phase={{PHASE}}; Date={{DATE}}; Timestamp (UTC)={{TIMESTAMP_UTC}}; Executed By={{EXECUTED_BY}}; Executed By ID={{EXECUTED_BY_ID}} or N/A; Execution Identity Source={{IDENTITY_SOURCE}}; Repository={{REPOSITORY}} or N/A; Branch={{BRANCH}} or N/A; Root Prompt={{ROOT_PROMPT}}; Skill Executed={{SKILL}} or N/A (standalone prompt); Sequence Position={{CURRENT_GLOBAL_SEQUENCE_POSITION}}; Total Prompts Executed={{TOTAL_PROMPTS_EXECUTED}}
```

After these standard fields, **document-type specific extension rows** are added per the template's `@METADATA` directive (see Section: EXTENSION FIELDS).

------------------------------------------------------------

## FIELD DEFINITIONS

| Field | Description | Mandatory | Source |
|-------|-------------|-----------|--------|
| Document Type | Name of the document type (e.g., "AECF Plan", "AECF Audit Code") | YES | Defined per template in `@METADATA` directive |
| AECF Number | Sequential document number within the TOPIC: `AECF_01`, `AECF_02`... | YES | Auto-resolved by SKILL_DISPATCHER |
| TOPIC | The resolved TOPIC for this execution chain | YES | User or auto-resolved |
| Scope | Files, modules, or directories in scope | YES | User or auto-resolved |
| Phase | AECF phase that generated this document | YES | Defined per template in `@METADATA` directive |
| Date | Generation date in `YYYY-MM-DD` format | YES | Auto-generated |
| Timestamp (UTC) | Precise generation timestamp in UTC (ISO-8601, e.g., `2026-02-20T15:21:09Z`) | YES | Auto-generated |
| Executed By | User who triggered the execution. Preferred: VS Code logged-in user. Fallback: OS/session user | YES | Auto-detected |
| Executed By ID | Stable user identifier when available (e.g., VS Code account/email/login). Use `N/A` if unavailable | YES | Auto-detected |
| Execution Identity Source | Source used to resolve execution identity (`vscode_account`, `os_user`, `manual`, `unknown`) | YES | Auto-detected |
| Repository | Active repository/workspace logical name if available | YES | Auto-resolved |
| Branch | Active git branch if available | YES | Auto-resolved |
| Root Prompt | Exact reference to the original prompt that initiated the chain | YES | Auto-tracked |
| Skill Executed | Skill name if from `skill_...` file. `N/A` for standalone prompts | YES | Auto-resolved |
| Sequence Position | Current position in the global execution sequence | YES | Auto-tracked |
| Total Prompts Executed | Total count of prompts executed so far in this chain | YES | Auto-tracked |

## HOW TEMPLATES REFERENCE THIS FILE

Each template includes a **compact `@METADATA` directive** (blockquote) instead of duplicating the full metadata table. Example:

```markdown
> **@METADATA** — Apply standard AECF metadata header from `templates/TEMPLATE_HEADERS.md`
> | Field | Value |
> |-------|-------|
> | Document Type | AECF Plan |
> | Phase | 00_PLAN |
```

The AI MUST:
1. Read this directive
2. Load the standard metadata block from this file (`TEMPLATE_HEADERS.md`)
3. Insert the compact `## METADATA` block as the final section in the generated document
4. Fill `Document Type` and `Phase` from the directive
5. Add any extension rows specified in the directive
6. Auto-resolve all other fields (TOPIC, Date, Timestamp, execution identity, repository context, etc.)

------------------------------------------------------------

## RULES

1. **Position**: `## METADATA` MUST be the LAST section in every generated document
2. **Format**: Compact single-line `key=value; key=value` format — no table, no bullet lists, no YAML frontmatter
3. **Completeness**: ALL standard fields must be present even if value is `N/A` or `TBD`
4. **No omissions**: Missing `Executed By`, `Executed By ID`, `Execution Identity Source`, `Timestamp (UTC)`, or `Date` → **DOCUMENT INVALID**
5. **Single source**: This file is the ONLY place where the metadata structure is defined
6. **Templates reference, not duplicate**: Templates contain `@METADATA` directives, NOT the full table
7. **Extensions after standard**: Document-type specific fields MUST appear AFTER the standard fields
8. **Future changes**: If this standard is updated, all NEW documents follow the new version. Previous documents remain immutable.
