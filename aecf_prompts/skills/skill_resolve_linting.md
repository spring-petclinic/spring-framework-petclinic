## Skill ID
`aecf_resolve_linting`

## Description
Produces a read-only resolution report for the repository quality profile that AECF should reuse before planning or implementation.

## Objective
Freeze and explain the deterministic linting and static-analysis profile so later phases and humans can reuse one canonical quality contract instead of re-deciding tools ad hoc.

## Supported Request Shapes

- Pre-plan resolution: "which quality profile applies before planning this change?"
- Pre-implement resolution: "which checks must this implementation satisfy?"
- Scope explanation: "why did AECF choose these tools for this repo or surface?"
- Reuse report: "summarize the current `static_analysis_profile` for this topic"

## Inputs

### Default behavior:
- If no extra scope is provided, explain the already resolved repository-level profile for the current topic

### Optional:
- `TOPIC` for the output folder or explicit report label
- User prompt hints about the intended surface or area of change
- Existing file lists, module hints, or semantic profile hints already captured by AECF state

## Mandatory Interpretation Rules

- The canonical machine-facing keys remain `static_analysis_matrix` and `static_analysis_profile`.
- If evidence is partial or confidence is not high, the report MUST say so explicitly instead of inventing certainty.
- The report MUST distinguish resolved checks from optional nice-to-have recommendations.
- The report MUST explain precedence between repo evidence, requested stack, semantic profile, and touched-surface hints.
- If the resolved profile implies code or config remediation, the skill MUST recommend `aecf_refactor` as the governed next step.

## Output Contract

The final report MUST contain these sections in English headers:

- `## METADATA`
- `## EXECUTIVE SUMMARY`
- `## ANALYSIS SCOPE`
- `## RESOLUTION INPUTS`
- `## RESOLVED STATIC ANALYSIS PROFILE`
- `## RESOLVED CHECK MATRIX`
- `## REUSE GUIDANCE`
- `## COVERAGE GAPS`
- `## RECOMMENDED NEXT ACTIONS`
